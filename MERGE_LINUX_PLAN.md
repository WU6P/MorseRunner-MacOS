# Plan: Merge Linux Support into MR_merge

## Goal
Add Linux (Ubuntu x86_64/ARM64) support to MR_merge, which already supports Windows and macOS.

## Current State

### MR_merge (target)
- **Windows**: `Main_win.pas`, `Log_win.pas`, `Ini_win.pas`, `MorseRunner.dpr/.dproj`, `VCL/*_win.pas`
- **macOS**: `mac/Main.pas`, `mac/Log.pas`, `mac/Ini.pas`, `mac/Globals.pas`, `mac/ContestFactory.pas`, `mac/PerlRegEx.pas`, `mac/GetDataPath.pas`, `mac/VCL/{SndOut,SndCustm,AudioBackend2,PermHint,VolmSldr,WavFile}.pas`
- **Shared**: All contest `.pas` files in root, `VCL/` common units, `Util/` parser units
- **LPI** (macOS): `MorseRunner.lpi` → unit path: `mac;mac/VCL;VCL;Util`
- **LPR**: `MorseRunner.lpr` → already has `{$ifdef unix}cthreads{$endif}`

### MR_Linux (source overlay)
- `linux/GetDataPath.pas` — XDG-compliant data/user paths
- `linux/VCL/SndOut.pas` — TAlSoundOut using PulseAudio ring-buffer backend
- `linux/VCL/AudioBackendPulse.c` — C backend (gcc compiled, linked via `{$L}`)
- `MorseRunner_linux.lpi` — Lazarus project file (GTK2 widgetset)
- `build_linux.sh` — Build script (gcc + lazbuild, auto-detects x86_64/aarch64)
- `install_deps.sh` — Dependency installer for Ubuntu
- `LINUX_PORT_README.md` — Full documentation

### Linux Reuses from mac/ (no changes needed)
- `mac/Main.pas` + `mac/Main.lfm` — LCL form (not Cocoa-specific)
- `mac/Log.pas`, `mac/Ini.pas`, `mac/Globals.pas` — LCL-generic
- `mac/ContestFactory.pas`, `mac/PerlRegEx.pas` — Pure Pascal
- `mac/VCL/WavFile.pas`, `mac/VCL/VolmSldr.pas`, `mac/VCL/PermHint.pas`, `mac/VCL/SndCustm.pas`

## What Linux Overrides (only 2 units)
1. **GetDataPath** → `linux/GetDataPath.pas` (XDG paths instead of .app bundle)
2. **SndOut** → `linux/VCL/SndOut.pas` (PulseAudio instead of CoreAudio)

Unit search path order for Linux LPI: `linux → linux/VCL → mac → mac/VCL → . → VCL → Util`

## Execution Steps

### Step 1: Copy linux/ overlay directory
```
MR_Linux/linux/GetDataPath.pas       → MR_merge/linux/GetDataPath.pas
MR_Linux/linux/VCL/SndOut.pas        → MR_merge/linux/VCL/SndOut.pas
MR_Linux/linux/VCL/AudioBackendPulse.c → MR_merge/linux/VCL/AudioBackendPulse.c
```

### Step 2: Copy Linux project file
```
MR_Linux/MorseRunner_linux.lpi → MR_merge/MorseRunner_linux.lpi
```

### Step 3: Copy build scripts
```
MR_Linux/build_linux.sh   → MR_merge/build_linux.sh
MR_Linux/install_deps.sh  → MR_merge/install_deps.sh
```

### Step 4: Copy documentation
```
MR_Linux/LINUX_PORT_README.md → MR_merge/LINUX_PORT_README.md
```

### Step 5: Verify prerequisites (already done in MR_merge)
- [x] Case-sensitivity fixes: `CQWWCW.txt`, `NAQPCW.txt`, `SSCW.txt`, `FDGOTA.txt` — already lowercase in MR_merge source
- [x] `MorseRunner.lpr` has `{$ifdef unix}cthreads{$endif}` — confirmed
- [x] `mac/VCL/SndCustm.pas` has `{$ifdef DARWIN}` guards for CoreAudio linkage — safe on Linux

### Step 6: No package_for_linux.sh needed
The `package_for_linux.sh` was designed for the old workflow (separate MR_Linux + MR_merge dirs).
In the merged repo, all files are already in one directory. Building on Linux just needs:
```bash
./install_deps.sh   # once
./build_linux.sh    # build
./MorseRunner       # run (launcher generated by build script)
```
